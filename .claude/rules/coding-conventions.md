# 编码规范

## Python 代码风格

- 标准 type hints，无过度抽象
- 三个相似行优于一个过早抽象
- 不添加功能标记、向后兼容 shim —— 直接改代码
- 默认不写注释。只有当 WHY 不明显时才加（隐藏约束、微妙不变量、特定 bug 的 workaround）

## 项目约定

- 不修改 `recipes/` 目录中的 YAML 文件（除非 goose 版本升级需要适配）
- 重大设计决策写入 `CLAUDE.md` 架构决策记录
- 新增功能必须更新 `.claude/status.md` 的开发状态追踪表格

## 安全

- 不在日志中泄露密码、accessKey、token、连接串凭据
- 子进程 env 必须最小化传递（仅必要变量）
- 路径必须验证在项目目录内（防穿越）
